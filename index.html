Simple REST API for Categories & Products

Base URL: http://localhost:3000

üìÅ Categories

GET /categories
Get all categories
Query: limit, offset, sortBy, order
GET /categories/:id
Get single category
POST /categories
Create new category
PUT /categories/:id
Update category
DELETE /categories/:id
Delete category (only if no products)
GET /categories/search-by-name/:name
Search categories by name

Category Fields:

name (required) - 2-50 characters, unique
quantity (required) - Non-negative number
description (optional) - Max 200 characters
isActive (optional) - Boolean, default: true
POST /categories { "name": "Electronics", "quantity": 50, "description": "Electronic devices and accessories" }

üì¶ Products

GET /products
Get all products
Query: limit, offset, sortBy, order
GET /products/:id
Get single product
POST /products
Create new product
PUT /products/:id
Update product
DELETE /products/:id
Delete product
GET /products/search-by-name/:name
Search products by name
GET /products/search-by-category/:categoryId
Get products in category

Product Fields:

name (required) - 2-100 characters
quantity (required) - Non-negative number
price (required) - Positive number
category (required) - Valid category ID
sku (optional) - Unique product code
image (optional) - Image URL
description (optional) - Product description
POST /products { "name": "iPhone 15", "quantity": 10, "price": 999.99, "category": "64f123456789abcd12345678", "sku": "IPH15-128GB" }

‚ö° Quick Reference

üí° Key Points:
  • All responses are in JSON format
  • Products must belong to a valid category
  • Categories with products cannot be deleted
  • Use pagination with limit and offset
  • Search is case-insensitive
# Test the API curl http://localhost:3000/categories curl http://localhost:3000/products # Create category first, then products curl -X POST http://localhost:3000/categories \ -H "Content-Type: application/json" \ -d '{"name":"Books","quantity":100}'
